# Chappie — Concepts & Design

A structured record of the ideas, decisions, and open debates behind Chappie.
These are living documents: Chappie is at the concept-and-philosophy stage, and
this is where we reason about *why*, not just *how*. Code is the ground truth;
these capture the intent the code is reaching for.

## The areas

1. [Thesis](01-thesis.md) — the bet: many small specialists + a good learning loop.
2. [Architecture](02-architecture.md) — the cognitive loop, attention, consensus.
3. [Memory](03-memory.md) — Complementary Learning Systems (working / episodic / semantic / parametric).
4. [Sleep & Dreaming](04-sleep-and-dreaming.md) — reliving, judging, recombining, wandering. *(the deepest area)*
5. [Cognition](05-cognition.md) — decisions, dual-process (reflex vs thinking).
6. [Development](06-development.md) — the life-cycle, embryo → real senses, the examined life.
7. [Open Questions](07-open-questions.md) — the philosophical debates we haven't settled.
8. [Valence & Drives](08-valence-and-drives.md) — good/bad as a learned curve; sleep as buffer reconciliation (Bitcoin-difficulty rhythm); boredom → creativity; reality-monitoring. *(supersedes the reward/energy sketches)*

## Reading guide

If you read one, read [Sleep & Dreaming](04-sleep-and-dreaming.md) — it's where the
architecture stops being bookkeeping and starts being *cognition pointed at itself*.

## Ground rules for these docs

- **Claims stay honest.** We mark what's built, what's stubbed, and what's aspiration.
- **Neuroscience is inspiration, not authority.** We borrow mechanisms where they earn
  their keep as engineering; we don't claim biological fidelity.
- **We name our metaphors.** "Global workspace", "dreaming", "hemispheres" are design
  frames, not assertions about experience. Where a metaphor makes a claim, it's flagged
  in [Open Questions](07-open-questions.md).
